Type a message and hit one of the buttons to fire up a toast alert or footer alert.

Footer alerts don't disappear automatically - but this can be easily implemented.

By default, Success and Info type toast alerts will disappear after 6 seconds. Warning and Error type toast alerts do not disappear.
Toasts stick together. The change in position won't happen until the visible toasts in the old position are removed.

 

In-situ alerts

  Success   Your day is off to a flying start

Can always be removed by clicking the close button (done with enableCloseButtosn in ux.js)

  Info   Elephants can not ride horses
  Warning   There is no money left in your bank account
  Error   Something terrible has happened

 

Icons

Default icons can be used for each state of alert.

  icon-alert-success   icon-alert-info   icon-alert-warning   icon-alert-error

 

Toast alerts are fired using common.alert

common.alert.warning('This is a warning', 'title of alert');

I found the fade out effect on close to be irritating, but there was no way with toastr params to have no "hide" animation, so I have modified the toastr source code slightly. Now the toasts disappear immediately when closed.

 

Footer Alerts are fired by emitting an event mtk.footerAlert on $rootScope

$rootScope.$emit('mtk.footerAlert', {type:type, msg: vm.alertMsg, title: vm.alertTitle});

 

Notes

The names are a bit confusing but I was unsure if both toasts and footer alerts will be used.

I could move the code that emits the event for the footer alert into common as well, that way both types could be called the same way. For example 

common.footerAlert.warning('This is a warning', 'title of alert');
common.toastAlert.warning('This is a warning', 'title of alert');
